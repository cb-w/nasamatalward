# نشر المتجر مجاناً حتى يظهر للجميع (دومين مجاني)

ستحصل على **رابط مجاني** مثل:  
`https://gift-perfume-store.onrender.com`  
أي شخص يفتح هذا الرابط يرى متجرك. لا تحتاج خبرة برمجة — فقط اتبع الخطوات بالترتيب.

---

## ما تحتاجه (مرة واحدة)

1. **حساب Gmail** (أو أي بريد)
2. **تحميل Git** على جهازك (لرفع المشروع):  
   → [تحميل Git لـ Windows](https://git-scm.com/download/win)  
   ثبّته ثم **أعد تشغيل الجهاز أو على الأقل أغلق وافتح من جديد نافذة الأوامر (Terminal / CMD)**.

---

## الخطوة ١: رفع المشروع إلى GitHub

1. ادخل إلى **[github.com](https://github.com)** وسجّل دخول (أو أنشئ حساباً جديداً).
2. اضغط **New** (أو **Create repository**).
3. **Repository name:** اكتب مثلاً: `gift-perfume-store`
4. اختر **Public**.
5. **لا** تضف Readme أو .gitignore (المشروع عنده بالفعل).
6. اضغط **Create repository**.

7. افتح **Terminal** (أو CMD أو PowerShell) وانتقل لمجلد المشروع:
   ```bash
   cd "C:\Users\omar joky\gift-perfume-store"
   ```
8. نفّذ الأوامر التالية **واحدة تلو الأخرى** (غيّر `YOUR_USERNAME` إلى اسم مستخدمك في GitHub):

   ```bash
   git init
   git add .
   git commit -m "متجر هدايا وعطور"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/gift-perfume-store.git
   git push -u origin main
   ```

   عند طلب **اسم المستخدم وكلمة المرور:**  
   - اسم المستخدم = اسمك في GitHub  
   - كلمة المرور = لا تستخدم كلمة مرور الحساب العادية. استخدم **Personal Access Token**:  
     من GitHub: **Settings** → **Developer settings** → **Personal access tokens** → **Generate new token**، اختر صلاحية **repo**، انسخ الـ Token واستخدمه مكان كلمة المرور.

بعد نجاح `git push` يصبح المشروع على GitHub.

---

## الخطوة ٢: نشر الموقع على Render (دومين مجاني)

1. ادخل إلى **[render.com](https://render.com)**.
2. اضغط **Get Started for Free** وسجّل دخول باستخدام **GitHub** (Sign in with GitHub).
3. بعد الدخول اضغط **New +** ثم **Web Service**.
4. في "Connect a repository" اختر المستودع **gift-perfume-store** (أو الاسم الذي استخدمته).  
   إن لم يظهر، اضغط **Configure account** واسمح لـ Render بالوصول للمستودعات ثم اختر المستودع.
5. املأ الحقول:
   - **Name:** اترك الافتراضي أو غيّره (مثلاً `my-gift-store`) — هذا سيظهر في الرابط.
   - **Region:** اختر الأقرب لك (مثلاً Frankfurt).
   - **Branch:** `main`
   - **Runtime:** `Node`
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Instance Type:** اختر **Free**.

6. اضغط **Create Web Service**.
7. انتظر بضع دقائق حتى ينتهي البناء والنشر (Build & Deploy).
8. عند ظهور **Live** أو رسالة نجاح، انسخ الرابط الظاهر فوق (مثل `https://gift-perfume-store.onrender.com`).

هذا الرابط هو **الدومين المجاني** لموقعك — أرسله لأي شخص وسيفتح المتجر.

- **المتجر للزبائن:** نفس الرابط (مثلاً `https://gift-perfume-store.onrender.com`)
- **لوحة التحكم:** نفس الرابط + `/admin` (مثلاً `https://gift-perfume-store.onrender.com/admin`)  
  كلمة المرور الافتراضية: `admin123` — غيّرها من الإعدادات بعد الدخول.

## ملاحظات مهمة

- **الخطة المجانية على Render:** الموقع قد "ينام" بعد حوالي 15 دقيقة بدون زيارات. أول زيارة بعد النوم قد تأخذ 30–50 ثانية ثم يعمل طبيعي.
- **البيانات (المنتجات والطلبات):** على الخطة المجانية قد تُفقد عند إعادة النشر أو إعادة التشغيل. إذا احتجت حفظاً دائماً لاحقاً يمكن إضافة قاعدة بيانات (نخبرك كيف عند الحاجة).
- **تحديث الموقع:** كلما عدّلت المشروع على جهازك نفّذ من نفس المجلد:
  ```bash
  git add .
  git commit -m "تحديث"
  git push
  ```
  وسيعيد Render النشر تلقائياً إذا كان مربوطاً بنفس المستودع.
